# 🚀 RobinX — Пошаговый гайд по запуску (для новичков)

Этот файл проведёт тебя от «ничего нет» до «токен живой, торгуется, награды капают».
Делай строго по порядку, не перепрыгивай. Если что-то непонятно — остановись
на этом шаге и спроси.

Обозначения:
- 🖥️ = делаешь в терминале (приложение «Терминал» на Mac)
- 🦊 = делаешь в кошельке (MetaMask или Rabby)
- 🌐 = делаешь в браузере

> ⚠️ ГЛАВНОЕ ПРАВИЛО: пока не сделан внешний аудит и ты не потренировался —
> **не заливай большие деньги**. Один раз ошибёшься с приватным ключом или
> адресом — деньги не вернуть. Сначала маленькие суммы.

## ЧАСТЬ 0. Что тебе нужно приготовить (30 минут)

### 0.1. Кошелёк-деплоер
Заведи **новый** кошелёк в MetaMask или Rabby (не тот, где твои личные деньги).
С него будешь деплоить. Запиши сид-фразу на бумагу, никому не показывай.

### 0.2. Приватный ключ этого кошелька
В MetaMask: три точки → «Данные учётной записи» → «Показать приватный ключ».
Это строка из 64 символов вида `0xabc123...`. **Это пароль от денег. Никогда
никому не отправляй, не вставляй на сайты.** Понадобится только в терминале.

### 0.3. Газ (ETH на Robinhood Chain)
Чтобы деплоить, нужен ETH именно **на Robinhood Chain**, а не на Ethereum.
- Купи/имей немного ETH в обычной сети (Ethereum или Arbitrum).
- Переведи его на Robinhood Chain через официальный мост.
  Инструкция и ссылка на мост: https://docs.robinhood.com/chain/connecting/
- Нужно **0.05–0.1 ETH** — этого хватит на все транзакции с большим запасом
  (газ там очень дешёвый).

### 0.4. WETH для ликвидности
WETH — это «обёрнутый» ETH. В пул ROBX кладётся против WETH, чтобы покупатели
сразу покупали за ETH (награды при этом считаются в USDG — это делает контракт сам). Сколько класть в пул — решаешь сам
(типичный мемный старт — от 5 000 долларов). Тоже должен лежать на кошельке
**на Robinhood Chain**.

### 0.5. Установить программы на компьютер (один раз)
Открой Терминал и выполни по очереди:

🖥️ Установить Foundry (инструмент для деплоя контрактов):
```bash
curl -L https://foundry.paradigm.xyz | bash
```
Потом закрой терминал, открой заново и выполни:
```bash
foundryup
```
Проверка, что установилось:
```bash
forge --version
```
Должна появиться строчка с версией. Если «command not found» — напиши мне.

## ЧАСТЬ 1. Деплой контрактов (15 минут)

### 1.1. Зайти в папку контрактов
🖥️
```bash
cd /Users/johnbuz/Desktop/PROJECTS/robinhood/contracts
```

### 1.2. Проверить, что тесты зелёные (страховка)
🖥️
```bash
forge test
```
Должно быть `28 passed`. Если да — контракты в порядке, идём дальше.

### 1.3. Прогнать деплой «на сухую» (без реальных денег)
Это репетиция: forge покажет, что будет делать, но НЕ отправит в сеть.
🖥️
```bash
forge script script/Deploy.s.sol --rpc-url https://rpc.mainnet.chain.robinhood.com
```
Если отработало без красных ошибок — можно деплоить по-настоящему.

### 1.4. Настоящий деплой
Вставь свой приватный ключ вместо `ТВОЙ_КЛЮЧ` (с `0x` в начале).
🖥️
```bash
forge script script/Deploy.s.sol \
  --rpc-url https://rpc.mainnet.chain.robinhood.com \
  --private-key ТВОЙ_КЛЮЧ \
  --broadcast \
  --verify \
  --verifier blockscout \
  --verifier-url https://robinhoodchain.blockscout.com/api
```

Что произойдёт автоматически:
- создастся токен **ROBX** (все 100 000 000 придут тебе на кошелёк),
- создастся **RewardDistributor** (казна),
- они свяжутся между собой,
- роутер станет reward-exempt (важно для безопасности),
- разрешатся акции-награды TSLA / NVDA / SPCX,
- код контрактов опубликуется в эксплорере (все смогут его читать — это плюс к доверию).

### 1.5. ЗАПИСАТЬ АДРЕСА ⚠️
В конце терминал напечатает две строки:
```
ROBX             : 0x..........
RewardDistributor: 0x..........
```
**Скопируй обе в заметки прямо сейчас.** Они нужны на каждом следующем шаге.
Дальше в гайде: `АДРЕС_ROBX` и `АДРЕС_КАЗНЫ`.

### 1.6. Проверить в эксплорере
🌐 Открой https://robinhoodchain.blockscout.com и вставь `АДРЕС_ROBX` в поиск.
Ты должен увидеть свой токен, 100M суплая, и зелёную галочку «Verified» на коде.

## ЧАСТЬ 2. Создать пару и добавить ликвидность на Uniswap (20 минут)

Сейчас токен существует, но его нельзя купить — нет пула. Создаём его.

### 2.1. Добавить Robinhood Chain в кошелёк
🦊 Если сети ещё нет — добавь вручную (MetaMask → Сети → Добавить сеть):
- Название сети: `Robinhood Chain`
- RPC URL: `https://rpc.mainnet.chain.robinhood.com`
- Chain ID: `4663`
- Символ валюты: `ETH`
- Эксплорер: `https://robinhoodchain.blockscout.com`

### 2.2. Добавить токен ROBX в кошелёк (чтобы видеть баланс)
🦊 В кошельке «Импорт токена» → вставь `АДРЕС_ROBX`. Увидишь свои 100M ROBX.

### 2.3. Открыть Uniswap на нужной сети
🌐 Зайди на https://app.uniswap.org → «Connect» → выбери свой кошелёк →
вверху справа переключи сеть на **Robinhood Chain**.

### 2.4. Создать пул
🌐 Раздел **Pool** (или «Positions») → **New position** / **Create pool**.
- Выбери версию **v2** (наш контракт заточен под v2).
- Первый токен: **WETH** (адрес `0x0Bd7D308f8E1639FAb988df18A8011f41EAcAD73`) — чтобы люди покупали за ETH, а не искали USDG.
- Второй токен: **ROBX** — вставь `АДРЕС_ROBX`. Uniswap спросит подтвердить
  импорт незнакомого токена — подтверди (это твой токен).

### 2.5. Задать стартовую цену и внести ликвидность
- Впиши, сколько ROBX и сколько USDG кладёшь. Соотношение = стартовая цена.
  Пример: 30 000 000 ROBX + 2 WETH → это и задаёт стартовую цену.
  (Помни план: в рынок идёт ~30% суплая, остальное у команды в вестинге —
  про вестинг отдельно, не сливай всё в пул.)
- Нажми **Approve WETH**, подтверди в кошельке (транзакция №1).
- Нажми **Approve ROBX**, подтверди (транзакция №2).
- Нажми **Add / Supply**, подтверди (транзакция №3). Готово — пул создан,
  токен теперь торгуется.

### 2.6. Узнать адрес пары (pair address)
🌐 В эксплорере открой `АДРЕС_ROBX` → вкладка **Token** → там будет пул, или
открой свою транзакцию добавления ликвидности и найди созданный контракт пары
(адрес вида `0x...`). Это `АДРЕС_ПАРЫ`. Скопируй.

### 2.7. Включить налог 4% (две транзакции из кошелька)
Теперь надо сказать токену: «этот пул — рынок, бери с него 4%».
Проще всего через эксплорер:
🌐 Открой `АДРЕС_ROBX` в blockscout → вкладка **Contract** → **Write** →
подключи кошелёк (тот же деплоер) и вызови:
1. `setMarketPair` → вставь `АДРЕС_ПАРЫ`, вторым полем `true` → Write.
2. Проверь, что `setRewardExempt` для роутера уже стоит (деплой-скрипт это
   сделал). Если сомневаешься — вызови `setRewardExempt` с адресом роутера
   `0x89e5DB8B5aA49aA85AC63f691524311AEB649eba` и `true`.

### 2.8. Контрольная покупка
🌐 С ЛЮБОГО другого кошелька купи ROBX на $5–10 на Uniswap.
Проверь: пришло 96% токенов, а 4% упало на `АДРЕС_КАЗНЫ` (посмотри баланс
казны в эксплорере). Если так — механика налога работает. 🎉

## ЧАСТЬ 3. Награды акциями (10 минут)

### 3.1. Проверить, есть ли пулы у акций
🌐 На Uniswap проверь, существуют ли пары **tTSLA/USDG**, **tNVDA/USDG**,
**tSPCX/USDG** (акции на старте торгуются через RFQ, пулов может ещё не быть).

- **Если пулы есть** — ничего не делаешь, награды в акциях уже включены
  (деплой-скрипт их разрешил).
- **Если пулов нет** — не страшно: контракт будет платить награды в USDG,
  пока пулы не появятся. На сайте честно пишем «выплаты в акциях — скоро».

Ничего вручную включать не надо — всё уже настроено при деплое.

## ЧАСТЬ 4. Кипер на Railway (20 минут)

Кипер — это бот, который каждые 30 минут нажимает «раздать награды».
Без него награды не будут распределяться сами.

### 4.1. Завести отдельный кошелёк для кипера
🦊 Новый кошелёк (НЕ деплоер). Закинь ему **0.01 ETH** на Robinhood Chain на
газ. У него нет никаких прав — только платить за вызовы. Возьми его приватный ключ.

### 4.2. Загрузить проект на GitHub
Если репозитория ещё нет:
🖥️
```bash
cd /Users/johnbuz/Desktop/PROJECTS/robinhood
git init && git add . && git commit -m "RobinX"
```
Создай пустой репозиторий на github.com, потом:
```bash
git remote add origin ССЫЛКА_НА_ТВОЙ_РЕПО
git push -u origin main
```

### 4.3. Создать сервис на Railway
🌐 Зайди на https://railway.app → войди через GitHub → **New Project** →
**Deploy from GitHub repo** → выбери свой репозиторий.

### 4.4. Указать папку кипера
🌐 В настройках сервиса (Settings) найди **Root Directory** и впиши `keeper`.
Railway сам поймёт, что это Node-проект, и запустит `npm start`.

### 4.5. Прописать переменные
🌐 Вкладка **Variables** → добавь по одной:
| Имя | Значение |
| --- | --- |
| `DISTRIBUTOR` | `АДРЕС_КАЗНЫ` |
| `ROBX` | `АДРЕС_ROBX` |
| `KEEPER_PRIVATE_KEY` | приватный ключ кошелька-кипера |
| `RPC_URL` | `https://rpc.mainnet.chain.robinhood.com` |

### 4.6. Запустить
🌐 Railway пересоберётся и запустит бота. Открой вкладку **Logs** — должно
писать `keeper 0x... · gas balance ...` и каждую минуту `not ready · next
epoch in Xm`. Каждые 30 минут будет `distribute()` — награды пошли. 🎉

Держи сервис включённым (тип **Service**, не Cron). Газ тратится по копейкам,
0.01 ETH хватает надолго.

## ЧАСТЬ 6. Безопасность (СДЕЛАТЬ ДО крупной ликвидности)

### 6.1. Передать владение контрактами на мультисиг
Пока владелец контрактов — твой единственный кошелёк-деплоер. Это риск
(потеряешь ключ — потеряешь управление; украдут ключ — украдут управление).
Заведи мультисиг (Safe, если развёрнут на чейне) и передай ему владение
обоими контрактами (функция `transferOwnership`).

### 6.2. Залочить ликвидность
LP-токены (доказательство твоей ликвидности) залочь в локер или сожги —
это публично проверяют, сильный сигнал доверия.

## Порядок одним взглядом

1. Приготовить: кошелёк, ETH на газ, USDG, установить Foundry (Часть 0)
2. `forge script … --broadcast` → записать 2 адреса (Часть 1)
3. Uniswap: создать пул ROBX/WETH, залить ликвидность (Часть 2)
4. `setMarketPair` → включить налог, контрольная покупка (Часть 2)
5. Проверить пулы акций / иначе выплаты в USDG (Часть 3)
6. Railway: залить кипер, прописать переменные, кинуть газ (Часть 4)
7. Вписать адреса в сайт, задеплоить (Часть 5)
8. Владение → мультисиг, залочить LP (Часть 6)
